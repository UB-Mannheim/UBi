# Konfigurationshinweise für sicheren Zoom-Betrieb

Für einen datensparsamen und möglichst sicheren Betrieb von Zoom werden folgende Einstellungen empfohlen:

## 🔒 Zugriffs- und Meeting-Sicherheit

- **Kennwort:** Für jedes Meeting sollte ein individuelles Kennwort gesetzt werden.
- **Teilnehmer-Mikrofon:** Die Mikrofone der Teilnehmenden sollten bei Beitritt stummgeschaltet sein. Der Host entscheidet dann, ob eine Aufhebung der Stummschaltung möglich ist.
- **Kamera/Mikrofon Standard:** Mikrofon und Kamera sollten standardmäßig, also bei Beitritt, ausgeschaltet sein.
- **Benutzername:** Die Möglichkeit, sich selbst während einer Sitzung umzubenennen, sollte aktiviert bleiben.

## ⚙️ Host-Kontrolle und Berechtigungen

- **Aufzeichnung:** Nur der Host sollte eine Aufnahme starten können.
- **Annotation:** Die Möglichkeit der Annotation sollte nur für den Host verfügbar sein.
- **Virtual Backgrounds:** Die Nutzung virtueller Hintergründe sollte erlaubt und sogar empfohlen werden.

## 💾 Datenschutz und Datenminimierung

Um den Datenschutz zu gewährleisten, sollten folgende Funktionen deaktiviert oder eingeschränkt werden:

- **Cloud-Aufzeichnung:** Das Speichern von Aufzeichnungen in der Cloud sollte unterbunden werden.
- **Chat-Speicherung:** Das Speichern des Chats sollte unterbunden werden.
- **Dateitransfer:** Das Übertragen von Dateien über Zoom sollte unterbunden werden.
- **Feedback:** Das automatische Feedback an Zoom sollte deaktiviert werden.
- **Datacenter Regions:** Die „Datacenter Regions“ sollten weitest möglich eingeschränkt werden; idealerweise nur für EU bzw. den Europäischen Wirtschaftsraum freigegeben sein.
