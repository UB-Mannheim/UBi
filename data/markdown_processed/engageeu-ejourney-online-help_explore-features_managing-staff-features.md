# Managing Staff Features in eJourney

This guide outlines the specific functionalities available to Receiving Staff (Host Institutions) and Sending Staff (Home Institutions) within the eJourney platform for managing student applications and nominations.

## 🧑‍🏫 Receiving Staff (Host Institution)

Receiving Staff manage the intake and processing of applications for students nominated by partner institutions.

### 1. Orientation and Programme Setup

During the Orientation phase, Receiving Staff prepare and configure the program details for publication and application management within the platform.

#### A. Uploading Programme Data (Data Hub)

Before a short programme can appear in eJourney, it must first be created and published in the **Data Hub**, which is a separate platform. Receiving Staff are responsible for:

- Uploading and maintaining the programme information in the Data Hub.
- Ensuring that all required details are complete and accurate.

#### B. Configuring Programme Settings in eJourney

Once the programme is published in the Data Hub, the data automatically transfers to eJourney. Receiving Staff can then access the programme via the **ENGAGE.EU Admin menu** and configure specific settings:

**Available Programme Settings:**

- **Capacity:** Maximum number of students who can be admitted.
- **Application Start Date:** The date (inclusive) from which students can submit applications.
- **Application End Date:** The last date (inclusive) for application submission.
- **Nomination Deadline:** The last date (inclusive) by which sending institutions must complete student nominations.
- **Admission Deadline:** The last date (inclusive) by which the host institution must confirm admissions.
- **Admission Acceptance Deadline:** The last date (inclusive) by which admitted students must accept their offer.

### 2. Application Monitoring

#### A. Application Summary Graph

During the application period, this visual summary shows the number of applications submitted from each partner university. This feature allows staff to:

- Monitor the distribution of applications across partner institutions.
- Identify universities with comparatively low application numbers.
- Adjust promotional or outreach efforts if necessary.

> **Important Note:** The graph only reflects applications currently in the system. Counts may decrease or drop to zero once applications are processed by the sending institutions. It is recommended to review this graph both during the application period and again after the application deadline but before the nomination phase begins.

### 3. Admission Overview (Post-Nomination)

The **Admission Overview** provides a consolidated view of all applicants put forward by their home institutions.

> **Important:** This feature is only relevant *after* sending institutions have completed the nomination process. Until nominations are finalized, no applications will be visible here.

Once available, the overview displays all applicants who have been:

- Nominated, or
- Placed on the waiting list by their home institution.

**Information Displayed per Applicant:**

- Programme ID and Title
- Applicant Name, Email Address, European Student Identifier (ESI)
- Matriculation Number and Birthdate
- Gender, Degree, and Degree Level (e.g., Bachelor, Master)
- Current Funding (provided by the sending institution)
- Application Score (provided by the sending institution)
- Current Application Status

The **Application Status** field allows Receiving Staff to filter applicants based on nomination or waitlisting status. Hosting institutions may also request the inclusion of further custom fields.

#### Downloading the Admission Overview

Receiving Staff can download the Admission Overview as an Excel file to facilitate the sending of admission offers. The exported file includes all data fields available in the overview, allowing staff to work offline, perform additional analysis, or share information internally.

> **Important:** The downloaded file **does not** include application documents (e.g., transcripts or motivation statements). These documents are only accessible to Sending Staff, as they are responsible for assessing eligibility and making nomination decisions.

## 🧑‍🎓 Sending Staff (Home Institution)

Sending Staff manage the process of selecting and nominating eligible students to host institutions.

### 1. Application Overview

The **Application Overview** provides a consolidated view of all applicants from the sending institution who have applied to programs. This feature supports the nomination process by allowing staff to review and manage applications before submitting final decisions.

The Application Overview displays all applicants who have applied through the platform. For each applicant, the following information is shown:

- Programme ID and Title
- Applicant Name, Email Address, European Student Identifier (ESI)
- Matriculation Number and Birthdate
- Gender, Degree, and Degree Level (e.g., Bachelor, Master)
- Current Funding
- Application Score
- Current Application Status

### 2. Downloading the Application Overview

Sending Staff can download the full **Application Overview** as a ZIP file for offline processing and record keeping.

**Contents of the ZIP File:**

1. An Excel file containing all data from the Nomination Overview.
1. Applicant-uploaded documents, including:
   - Motivation Statements
   - Transcripts of Records

**Folder Structure:**

- `Programmes` – Separate folders for each short programme (named by Programme ID).
- `Applicants` – Within each programme folder, separate folders for each applicant (named using `ESI_last name`).
- `Documents` – Each applicant folder contains their uploaded Motivation Statement and Transcript of Records.

This feature allows Sending Staff to review application data and supporting documents offline, preparing and managing nominations efficiently.

> **Important:** It is essential to extract the ZIP file, as the Motivation Statements and Transcripts of Records cannot be opened otherwise.
